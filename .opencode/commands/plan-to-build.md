description: 将 Plan 方案写入任务文档，用户确认后进入 Build
agent: build
---

在 Plan 讨论完成后、Build 执行前调用。只写文档，不写代码。

## 1. 读取当前任务

`read tasks/doing/task-*.md` 了解已有记录。

## 2. 提取 Plan 决议

从会话中梳理以下内容:

- 最终方案 (一句话)
- 关键决策 (做了哪些选择、放弃了哪些选项)
- 执行步骤 (按顺序)
- 涉及文件
- 风险 / 待确认事项

## 3. 写入任务文档

在任务文件中追加 `## Plan 决议` 章节, 记录上述内容。
如方案涉及知识库更新, 同步更新 `docs/doc-*.md`。

## 4. 输出摘要

```
## plan-to-build 摘要

| 项目 | 内容 |
|------|------|
| 任务 | task-yyyy-mm-dd-xxx |
| 方案 | 一句话说明 |
| 涉及文件 | src/a.js, packages/@xxx/ |
| 执行步骤 | 1) ... 2) ... 3) ... |
| 风险 | 依赖 xxx 尚未验证 |
```

## 5. 确认

用 question 工具:

- **确认** → "方案已确认，进入 Build 阶段"
- **返回 Plan** → "返回 Plan 继续讨论"
