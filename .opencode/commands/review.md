## 1. 收集变更

```bash
git diff --stat
git diff
git submodule status
```

## 2. 审查维度

### 约定合规性
- 项目约定中禁止直接修改的共享区域是否被改动?
- 涉及平台特定 API 时是否使用了兼容层?
- 文件命名/代码风格是否符合项目规范?
- 是否使用了项目禁止的语言特性?

### 完整性
- Plan 决议中的步骤是否全部完成?
- 新增模块的导出/配置是否完整?
- 错误处理和边界情况是否覆盖?

### 安全/风险
- 是否引入了新的依赖或 API 而未适配项目环境?
- 是否有硬编码的路径或密钥?
- 是否有潜在的性能或安全问题?

### 误改动排查
- 是否有不属于本次任务的无关改动?
