---
description: 按 Plan 决议逐步执行, 每步验证
agent: build
---

## 1. 读取 Plan 决议

读取当前进行中任务文件, 提取 `## Plan 决议` 的执行步骤.

- **无 Plan 决议** → 提示用户: "此任务尚无 Plan 决议, 请先用 /plan 讨论方案, 再用 /design 写入"
- **有 Plan 决议** → 进入执行阶段

## 2. 按步骤执行

对每步:
1. 输出当前步骤描述和预期结果
2. 执行修改 (读文件 → 编辑/写入)
3. 步骤内自检: 是否达成该步骤目标
4. **关键步骤后运行 /check** 验证
5. **阶段性完成时运行 /checkpoint** 中转保存

## 3. 遇到阻塞

分析原因后询问用户:
- **返回 Plan** → "阻塞需重新分析, 是否回到 /plan?"
- **跳过此步** → 记录跳过原因, 继续下一步
- **调整方案继续** → 记录调整内容, 继续 (但仍建议通知 plan phase 的变化)

## 4. 全部步骤完成

```
## Build 完成摘要

### 已完成
- 步骤 1: xxx ✅
- 步骤 2: xxx ✅
- ...

### 变更文件
- src/a.js — 修改
- src/b.js — 修改

### 建议后续
- /x-review — 提交前代码审查
- /commit — 正式提交成果
```
